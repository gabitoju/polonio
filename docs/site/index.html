Polonio v0.1.0

Single-binary templating language for modern CGI-friendly HTML workflows.

Build HTML the 2005 way, with 2026 tooling.

Polonio mixes straightforward HTML with `<% %>` code blocks, `$var` interpolation, a pragmatic interpreter, and a runtime that can jump between CLI rendering and CGI automatically. Everything ships as a self-contained C++17 executable.

The language spec is frozen for v0.1.0, so you can count on consistent syntax, a predictable runtime, and a growing library of built-in helpers.

What is Polonio?

Polonio started as a CGI experiment and now delivers a modern lexer, Pratt parser, tree-walking interpreter, template renderer, and HTTP runtime in one binary. Templates stay readable because HTML and Polonio code coexist without ceremony, so designers can skim markup while developers wire in loops, includes, and functions.

Core pillars:

  • HTML-first templates: literal HTML interleaved with `<% ... %>` and `<% echo expr %>` blocks.
  • 7 fundamental types (`null`, `bool`, `number`, `string`, `array`, `object`, `function`) with lexical scoping and closures.
  • A rich standard library spanning string, array, object, math, date, output, and HTTP helpers.
  • CGI awareness: `_GET`, `_POST`, `_COOKIE`, and `_SERVER` superglobals plus response helpers like `http_status`.

Quick Start

1. Build the CLI

Polonio uses a simple Makefile. Build and run tests from the project root:

$ make
$ make test

The binary lands at build/polonio.

2. Hello World

Create hello.pol:

<% var name = "World" %>
<h1>Hello $name!</h1>

Render it:

$ ./build/polonio run hello.pol

3. Start the Dev Server

The polonio serve experience is tracked in Milestone 10. The CLI already parses the command, but the server is not implemented yet.

$ ./build/polonio serve --port 8080
serve: not implemented yet
`serve` remains planned work (docs/FEATURE_QUEUE.md still lists M10 items as TODO). Keep the command around for parity with the future workflow.

CLI Reference

Command Description
polonio help Show usage, commands, and a reminder that the dev server is pending.
polonio version Print the frozen version string (0.1.0).
polonio run <file.pol> Render a template. If the file contains `<%`, the template engine runs; otherwise Polonio executes pure code.
polonio <file.pol> Shorthand for run.
polonio --dump-ast <expr> Developer helper that prints the Pratt parser AST for the inline expression.
polonio serve [--port N] [--root DIR] Reserved for the dev server milestone. Today it prints “not implemented yet” and exits with failure.
