Examples & Cookbook

Practical templates that mirror the shipping interpreter and built-in functions.

Hello World

The classic starter shows literal HTML, a `<% var %>` declaration, `$name` interpolation, and a spanning `<% if %>` block. Because interpolation is raw, call html_escape (alias of htmlspecialchars) whenever untrusted data is displayed.

<% var name = _GET["name"] or "World" %>
<main class="hello">
  <h1>Hola $name!</h1>
  <% if not _GET["name"] %>
    <p>Append <code>?name=Ana</code> to the query string.</p>
  <% else %>
    <p>You sent: <% echo html_escape(_GET["name"]) %></p>
  <% end %>
</main>
Output: <h1>Hola Ana!</h1> *(when ?name=Ana is provided)*

Guestbook

Polonio’s CGI mode populates `_GET`, `_POST`, `_COOKIE`, and `_SERVER`. This guestbook stores messages in-memory (per request) and demonstrates loop constructs, push, and date helpers.

<%
var entries = []
if _POST["message"] %>
  <% push(entries, {
    "body": _POST["message"],
    "stamp": date_format(now(), "YYYY-MM-DD HH:mm")
  }) %>
<% end %>
<form method="post">
  <textarea name="message"></textarea>
  <button>Sign</button>
</form>
<ul class="entries">
  <% for entry in entries %>
    <li>
      <strong><% echo entry["stamp"] %></strong>
      <p><% echo html_escape(entry["body"]) %></p>
    </li>
  <% end %>
</ul>
For persistent storage you would wire entries to a file or database. Polonio keeps arrays and objects alive for the request and reinitializes them on the next run.

Todo Dashboard

Arrays of objects plus for i, item in items make simple dashboards ergonomic.

<%
var todos = [
  {"title": "Ship docs", "done": false},
  {"title": "Polish lexer", "done": true}
]
%>
<table class="todo">
  <tr><th>#</th><th>Task</th><th>Status</th></tr>
  <% for i, task in todos %>
    <tr>
      <td><% echo i + 1 %></td>
      <td><% echo html_escape(task["title"]) %></td>
      <td>
        <% if task["done"] %>
          <span class="done">Complete</span>
        <% else %>
          <span class="pending">In progress</span>
        <% end %>
      </td>
    </tr>
  <% end %>
</table>
Tip: Combine with concat to merge todos loaded from multiple includes.

Includes & Partials

include "file.pol" executes another template using the current interpreter state. Paths are resolved relative to the including file, with a depth limit of 50 and cycle detection (see template_renderer.cpp).

<% var page = "docs" %>
<% include "partials/header.pol" %>
<section>
  <h2>Content</h2>
</section>
<% include "partials/footer.pol" %>
Included templates inherit the same scope, so variables declared before an include remain available (and mutable) inside the partial.

Common Patterns

  • Formatting helpers: Use date_format(now(), "YYYY-MM-DD") or date_add_days for schedule pages.
  • Safe output: Prefer html_escape over htmlspecialchars (they are equivalent) for clarity, and nl2br when preserving multiline text.
  • Objects as dictionaries: Pair set, get, and has_key to emulate associative arrays.
  • Debugging: debug(value) writes a single-line description to stderr without disrupting page output.
  • Loop guards: The interpreter aborts loops after 100,000 iterations; chunk long work or pre-filter arrays first.

Original prueba.html

The repository’s history/prueba.html captures the 2005 prototype: ISO-8859-1 markup, inline `/* ... */` comment experiments, and the uppercase VAR declaration that lived before the current var keyword.

<h1>Prueba</h1>
Hola/*<br>
Ho#la
<% VAR pepe >
$pepe
  • Polonio still strips `/* ... */` comments while scanning template text, a direct nod to that file.
  • Modern Polonio expects the lowercase var keyword; the legacy uppercase spelling is kept in the history file only.
  • Encoding quirks (ISO-8859-1 headers, repeated meta tags) are preserved in the history folder for posterity.
Today’s interpreter normalizes everything to UTF-8 and requires `<% var name = "value" %>`—convert historic `VAR` declarations before running them.
