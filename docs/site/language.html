Language Reference

Summary of the Polonio v0.1.0 grammar, semantics, and interpreter behavior.

Template Structure

Templates mix literal HTML (emitted verbatim) with embedded code blocks. Files usually carry the .pol extension, but any extension works when you call polonio run.

Code blocks

<%
var title = "Docs"
if _SERVER["REQUEST_METHOD"] == "POST"
  echo "Posted"
end
%>

Interpolation + inline echo

<h1>$title</h1>
<p>2 + 3 = <% echo 2 + 3 %></p>
<p>Literal $: write "$$" in text.
  • `<% ... %>`: Multiline code, including control flow, function declarations, and includes.
  • `$var`: Works only in HTML/text segments. `null` becomes the empty string, and no escaping happens automatically.
  • Inline `<% echo expr %>`: Shortcut inside HTML where full blocks would be awkward.
Both the lexer and the template scanner support `/* ... */` block comments. Inside HTML segments those comments are stripped before code runs, mirroring the original prueba.html behavior.

Types

Polonio has seven runtime types implemented in Value (src/polonio/runtime/value.cpp).

TypeDescription
nullDefault value for uninitialized variables and missing function arguments.
booltrue / false.
numberDouble-precision floating point (all integers are represented as doubles).
stringUTF-8 text with " or ' quoting plus \n, \t, \", \' escapes.
arrayOrdered list (std::vector) accessed with numeric indices.
objectString-keyed map storing arbitrary values.
functionUser-defined or builtin function values with lexical closures.

Variables

<%
var name
var count = 2
name = "Ana"
count += 3
count ..= " total"
%>
  • `var name`: Declares a variable initialized to null.
  • `var name = expr`: Declares and initializes.
  • Assignment: `=`, compound math (`+= -= *= /= %=`) and concatenation (`..=`) operators are implemented in the interpreter.
  • Legacy casing: Only lowercase var is recognized; the uppercase VAR from archived templates is no longer tokenized as a keyword.

Operators

Expressions follow Pratt-precedence parsing (parser/parser.cpp). Everything is right-associative for assignments, and function calls/indexing bind tight.

Precedence (high → low)Operators
Call / Indexfn(), array[index], object["key"]
Unary-expr, not expr
Multiplicative*, /, %
Additive+, -
Concatenation..
Comparison<, <=, >, >=
Equality==, !=
Logical ANDand
Logical ORor
Assignment=, +=, -=, *=, /=, %=, ..=
Comparisons do not coerce types. Only `number == number`, `string == string`, etc. return true; mixed types compare as false.

Control Flow

If / Elseif / Else

<%
if user["role"] == "admin"
  echo "Welcome"
elseif user["role"] == "editor"
  echo "Hi"
else
  echo "Guest"
end
%>

Loops

<%
for i, item in items
  echo i .. ": " .. item
end

while total < 10
  total += 1
end
%>
  • for/in: Iterates arrays and objects. Use for value in array or for index, value in array. Objects yield key + value pairs.
  • while: Evaluated each loop. Once the condition becomes false or the iteration cap is hit the loop exits.
A global iteration guard (kLoopIterationLimit = 100000 in interpreter.cpp) applies to for and while. Hitting the limit raises “loop limit exceeded”.

Functions

<%
function greet(name)
  return "Hello " .. name
end

function counter()
  var value = 0
  return function()
    value += 1
    return value
  end
end
%>
  • Functions capture their surrounding environment; returning a nested function produces a closure.
  • Parameters default to null when callers omit arguments (see eval_call).
  • return exits the nearest function. If no value is provided, null is returned.
  • Named functions are self-recursive because the interpreter binds the function name inside its environment.

Includes

include "partial.pol" streams another template using the same interpreter and environments.

<% include "partials/nav.pol" %>
  • Paths are resolved relative to the including file’s directory.
  • The renderer canonicalizes paths and enforces a depth limit of 50. Circular chains raise “include cycle detected.”
  • Variables defined before the include remain readable and writable inside the partial.
  • Includes are disabled outside template rendering (you will see “include not supported here” if no include callback is registered).

Truthiness

The interpreter uses Value::is_truthy:

ValueTruthy?
nullfalse
falsefalse
truetrue
numberfalse if 0.0, otherwise true.
stringfalse if empty, otherwise true.
array / objecttrue (even if empty) because they store shared pointers.
functionAlways true.

Scope & Closures

Polonio uses lexical environments (Env):

  • Each function stores a pointer to the environment active at definition time (closure in FunctionValue).
  • Loops create child environments when binding for indices/values, so inner assignments don’t leak out unless you write to outer variables explicitly.
  • include does not create a new environment; partials share the same scope to keep template variables fluid.
  • Globals live in the interpreter’s root environment, which is preloaded with builtins.
Use helper functions to hide state when needed. Large templates often wrap logic in function render_item(item) and call it immediately to avoid polluting globals.
